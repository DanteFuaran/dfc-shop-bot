# Changelog

## [0.1.61] - 2025-02-02

### Fixed
- **Критическое исправление**: изменен Remnawave webhook URL на внутренний Docker адрес
- Убран JavaScript workaround - теперь используется NATIVE webhook от Remnawave
- Кнопки подключения теперь используют прямые ссылки на subscription.url
- Удалены endpoint'ы `/connect/` и `/subscription/` с HTML/JS
- Реализация теперь полностью соответствует snoups/remnashop

## [0.1.60] - 2025-02-02

### Fixed
- Исправлено: кнопка "Страница подписки" теперь работает идентично кнопке "Подключиться"
- Endpoint `/api/v1/subscription/` теперь использует ту же логику отслеживания устройств что и `/api/v1/connect/`
- Уведомления о добавлении устройств приходят при нажатии на любую из кнопок подключения

## [0.1.59] - 2025-02-02

### Changed
- JavaScript страницы подключения теперь всегда вызывает `/notify-device-connected` endpoint
- Бэкенд определяет новые устройства через сравнение HWID-set в Redis
- Улучшена логика определения новых устройств при повторном добавлении

## [0.1.58] - 2025-02-02

### Fixed
- Исправлена ошибка DI в endpoint `/user-devices/` (изменен `get_devices()` на `get_devices_user()`)

## [0.1.57] - 2025-02-02

### Added
- Реализовано отслеживание HWID устройств в Redis с TTL 30 дней
- Endpoint `/notify-device-connected/` для отправки уведомлений о новых устройствах
